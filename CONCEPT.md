# Let's Architect! — コンセプトドキュメント

若手社員がシステム・サービス設計を「判断の追体験」で学ぶ、ブラウザ完結型のケース判断アドベンチャーゲーム。

## 一言コンセプト

> 新人アーキテクトとして架空の案件に挑み、5〜10分のエピソードで「設計の判断力」を鍛えるゲーム。

正解を暗記させるのではなく、**曖昧な要望・限られた予算・トレードオフ**の中で「なぜその判断をするのか」を体験させる。

## ターゲットと利用シーン

- **ターゲット**: 入社1〜3年目のエンジニア・IT職。設計の実務経験はまだ浅い層。
- **利用シーン**: 昼休みや研修の合間などのスキマ時間。1エピソード5〜10分で完結。
- **前提知識**: 「サーバー」「DB」が何かは知っているレベル。専門用語は作中で先輩キャラが解説。

## ゲームの骨格

### 世界観

プレイヤーは架空のSI/サービス企業「(社名未定)」に配属された新人。
メンター役の**先輩キャラ**と一緒に、顧客案件や自社サービスの設計を任されていく。

### 1エピソードの流れ(5〜10分)

```
1. 案件ブリーフィング   … 顧客・上司から曖昧な要望が来る
2. ヒアリングパート     … 質問を選んで要件を引き出す(聞き漏らすと後で困る)
3. 設計判断パート       … 2〜4回の分岐選択(構成・技術・進め方)
4. 結果パート           … 選択の帰結がストーリーで描かれる(リリース成功/障害/炎上…)
5. 振り返り(デブリーフ) … 先輩が判断のポイントと実務での原則を解説
```

### 評価軸(パラメータ)

選択ごとに複数の軸が変動し、「万能の正解はない」ことを体感させる。

| 軸 | 意味 |
|---|---|
| 顧客満足 | 要望・期待に応えられたか |
| システム品質 | 可用性・性能・保守性 |
| 予算/納期 | コストと時間を守れたか |
| 信頼(チーム/顧客) | ヒアリングや説明の仕方で変動 |

エピソード終了時に軸ごとの結果+総合ランク(S/A/B/C)。選択によって複数の結末があり、**周回して別ルートを試したくなる**設計にする。

### 学習領域とエピソード構成

シーズン(章)ごとにテーマを変え、要件定義〜アーキテクチャまでを横断する。

| シーズン | テーマ | エピソード例 |
|---|---|---|
| 1. 要件を聞く | 要件定義・非機能要件 | 「絶対に落ちないシステムにして」と言われたら / SLAって何%あればいいの |
| 2. 形にする | アプリ設計・API設計 | 画面から考えるかデータから考えるか / 使いにくいAPIが生まれる瞬間 |
| 3. 支える | インフラ・アーキテクチャ | アクセス急増の乗り切り方 / DBが単一障害点だった話 |
| 4. 揺さぶられる | トレードオフ総合演習 | 予算半分と言われたら何を削るか / 納期優先 vs 品質優先の修羅場 |

トレードオフ思考は独立した章にもしつつ、全エピソードの選択肢設計に共通軸として織り込む。

## 学習効果のための設計原則

1. **選択肢に「もっともらしい罠」を入れる** — 明らかな不正解は置かず、若手がやりがちな判断(過剰設計、確認せず着手、言われた通りに作る)を混ぜる。
2. **失敗を安全に体験させる** — 悪い選択をしても即ゲームオーバーにせず、障害や手戻りという「結果」を見せてから振り返りにつなげる。
3. **デブリーフで実務に接続する** — エピソード末尾で「現実ではこういう場面」「使える質問・考え方」を先輩キャラが1〜2分で解説。
4. **用語はその場で学べる** — 本文中の専門用語はタップで用語集ポップアップ。

## 技術方針(GitHub Pages 前提)

- **完全静的サイト**: バックエンドなし。ビルド成果物を GitHub Pages に配置。
- **シナリオはデータ駆動**: エピソードを JSON/YAML で記述し、エンジンと分離。→ シナリオ追加が非エンジニアでも可能になり、社内で育てられる。
- **進捗保存は localStorage**: クリア状況・ランク・バッジをローカル保存。アカウント不要。
- **スマホ対応**: スキマ時間利用を想定しレスポンシブ必須。
- 実装スタックは後続の技術設計で決定(素の TS + Vite 程度の軽量構成を想定)。

## スコープ外(初期リリースでは作らない)

- ランキング・対戦などサーバーが要る機能
- 自由記述の設計図エディタ(採点が難しい)
- 認証・ユーザー管理

## MVP(最初に作る範囲)

1. ゲームエンジン(シナリオ再生・分岐・パラメータ・結果画面・localStorage 保存)
2. シーズン1「要件を聞く」からエピソード3本
3. タイトル画面+エピソード選択画面+用語集
4. GitHub Pages への自動デプロイ(GitHub Actions)
