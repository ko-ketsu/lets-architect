# ゲーム全体エンディング「Let's Architect!」コンセプト

2026-07-07 起案。S4 完了を受けたゲーム全体の締めコンテンツ(2026-07-05 ユーザー指示)。

## 位置づけとトーン

- **ハッピーエンドではなく「序章の終わり」**。4シーズンは見習い期間であり、物語はアオイ先輩の支援なしで judgement する日々の幕開けで終わる。
- 全12話 S ランク達成者だけが見る**ご褒美コンテンツ**。ただし報酬は称賛ではなく「独り立ちの宣告」。
- ゲームタイトル「Let's Architect!」を最後にもう一度掲げ、「これからのあなた」への命令形として意味を反転させて幕を引く。

## 解放条件

- **全12エピソードの bestRank が S** で解放。localStorage の既存データで判定するため、**現行版で既に全Sのユーザーも即座に解放される**(遡及)。
- エピソード選択画面の最下部に「Final Episode」セクションを常設:
  - ロック中: グレーのカードで「全エピソードで S ランクを達成すると解放」+ 現在の達成数(例: S × 9/12)。周回動機の可視化を兼ねる。
  - 解放後: 通常カードとしてプレイ可能。何度でも再鑑賞できる。ランクバッジは出さない(ランク判定がないため)。クリア済みマークのみ。

## 体験仕様(通常エピソードとの差分)

| 項目 | 通常エピソード | エンディング |
|---|---|---|
| パラメータ4軸 | あり(ヘッダーメーター常時) | **なし**(メーター非表示) |
| choice のフィードバックカード | 毎回必ず出る | **出ない**(下記「最後の問い」参照) |
| ランク判定・リザルト画面 | あり | **なし**。代わりに称号画面 |
| 所要時間 | 5〜10分 | 5分程度 |
| glossary | あり | なし(新しい用語を出さない) |

データは `data/episodes/final.json`。既存スキーマのサブセット(scene / choice / ending 相当)に、エピソード直下の識別フィールド `"finale": true` を追加。engine / ui はこのフラグで params・ランク・フィードバックカードの処理をスキップする。smoke-test は final.json を別枠で検証(参照整合+通しプレイで終端到達。ランク assert・check-routes の対象外)。

## シナリオ構成(3幕)

### 第1幕: 静かな夜のオフィス — アオイの報告

桜井案件(s4e3)の熱が冷めた頃の残業終わり。アオイから「話がある」。**新規事業の立ち上げ専任として異動する**ことが決まった(桜井案件の実績で声が掛かった、という前向きな理由。湿っぽくしない)。

手帳をめくりながら、これまでの案件を2〜3件だけ軽く振り返る(マルカワ食品・シートピア等。カーテンコールはこの圧縮版のみ。12件並べない)。そして本題:

> アオイ「最近気づいたんだ。私がコメントする前に、あなた、もう決めてるでしょ。理由まで揃えて。」
> アオイ「実務ではね、選んだ直後に答え合わせをしてくれる人は隣にいない。結果は数か月後に、障害やユーザーの声になって返ってくる。……でも、あなたはもう大丈夫。自分で決断できるよ。」

**トーン規則(最重要)**: アオイは自分の功績を一切口にしない。「私が支えたから」「私のおかげ」系の台詞は禁止。プレイヤーの成長を**観察した事実**として語り、背中を押す。上からの評価(「合格」「認めてあげる」)も禁止。

### 第2幕: 最後の問い — フィードバックのない選択

そこへ新しい相談のメールが届く。s1e1 を思わせる無茶な要望の再来(例: 「絶対に落ちない、絶対に漏れない、絶対に間に合うシステムにしてほしい」— 3つの「絶対」でエスカレートさせ、初心を思い出させつつ少し笑わせる)。

アオイ「最初の一手、どうする?」— ここに**エンディング唯一の choice** を置く:

- 選択肢は3つ。**すべて有効で、正解も優劣もない**(effects なし・flags なし)。判断対象は「この案件にどういう構造的アプローチで入るか」の初動(原則Gを踏襲した"らしさ"の担保。例:「絶対」の中身をSLAに翻訳しに行く / 落ちてはいけない経路を特定しに行く / 制約の優先順位を先に握りに行く)。
- 選んでも**フィードバックカードが出ない**。パラメータも動かない。1拍の無音(ナレーション「……」等の間)を置いてから、アオイが笑う:

> アオイ「ほらね。もう選べる。私の感想、要らないでしょ。」

12話ずっと選択の直後に返ってきたフィードバックカードが**初めて出ない**ことで、支援の不在をメカニクスの欠落として体感させる。これがこのエンディングの核。

### 第3幕: 幕開け — タイトルドロップ

アオイの去り際の一言:

> アオイ「答え合わせは、現場がしてくれる。」

エピローグ。後日、主人公が一人でそのクライアント先のドアの前に立つ。一呼吸。

> ナレーション「ここまでは、序章。」

→ 画面転換してロゴ **「Let's Architect!」** をタイトルドロップ(専用演出。フェードイン等は控えめに)。

### 称号画面(リザルトの代わり)

- 称号 **「Architect」** を大きく表示。到達称号(Master 等)ではなく、「見習いが終わり、今日から肩書きが付く」という含み。
- 添え書き: 全12エピソード S ランク達成の表示。
- ボタン: 「タイトルへ戻る」。

## 執筆ルール

- 第1幕のトーン規則(功績を伏せる・観察として語る・背中を押す)を全編に適用する。
- 別れを湿っぽくしない。アオイは最後まで頼れる・少し辛口のまま。感傷的な長台詞・回想の連発は禁止。
- s4e3 debrief の締め「絞る力は、ちゃんとあなたの中に積み重なってる」がこのエンディングへの導線。矛盾しないこと。
- 新しい専門用語を出さない。学びの総括はやるが講義はしない(1〜2文の言い換えまで)。
- choice の3択は文長・具体度を揃える(優劣がないことが文面からも伝わるように)。
